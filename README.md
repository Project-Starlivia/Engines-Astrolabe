## Strune — ビューア

知識のネットワークを、マインドマップのような感覚で探索するための表現手法とビューアです。

特定の1ノードを中心に置き、そのノードが参照しているノードを右側に、参照されているノードを左側に配置します。周辺のノードをクリックすると、そのノードが新たな中心となり視点が移動します。これを繰り返すことで、ネットワーク全体を自由に探索できます。

サンプルとして、ゲーム開発技術の知識マップ(エンジン・言語・プラットフォーム・ミドルウェア・作品など 109 ノード)を同梱しています。

## ファイル構成

```
README.md            このファイル
data.json            データセット本体(source of truth)
strune.schema.json   データ形式の JSON Schema
index.html           ビューアのマークアップ(Vite エントリ)
src/main.ts          ビューアのロジック(data.json を import)
src/style.css        スタイル
src/types.ts         データ型(schema 由来)
vite.config.ts       単一HTML出力の設定(vite-plugin-singlefile)
scripts/bundle.ts    dist/index.html → strune.html へ複製
strune.html          ビルド成果物(全部入り・単体で file:// 動作)
```

`strune.html` はビルド成果物です。データやビューアを編集したら `bun run build` で再生成してください(開発中は `bun run dev` で HMR が効きます)。

## データ形式

Strune は、ノード(最小の知識単位)と有向エッジ(向きを持つ参照関係)からなるネットワーク構造を対象とします。

ノードのコアは 2 フィールドだけです。

```json
{
  "id": "unity",
  "references": ["game-engine", "csharp"]
}
```

- **id** — ノードの一意な識別子。人間が読める文字列を推奨します。
- **references** — このノードが参照するノードの id の集合。各要素が「自ノード → 参照先ノード」という有向エッジになります。

データセット全体は、ノードを集めた JSON 配列です。配列で表現しますが、定義上、ノードの順序に意味はありません。順序に依存した挙動は仕様として保証されません。references の順序・重複についても同様です。

### 拡張フィールド

コア(id / references)を保ったまま、任意のフィールドを自由に追加できます。追加フィールドの扱いはレンダラに委ねられます。同梱データセットでは次の 2 つを使用しています。

- **label** — 表示名。なければ id がそのまま表示されます。
- **description** — ノードの短い説明文。ビューアの中央カードとツールチップに表示されます。

## 運用規約(同梱データセット)

データセットを育てるにあたっての取り決めです。データ形式の仕様ではなく、このデータセットのローカルルールです。

1. **裸参照のデフォルト意味** — 修飾なしの参照は、その文脈で最も自然な関係を意味します。
   - エンジン → 言語: その言語で公式にゲームロジックを書ける
   - エンジン → プラットフォーム: そのプラットフォームへビルドできる
   - エンジン/言語 → 企業・個人: それが開発元である
   - 機能ノード(2D Rendering / Physics など)への参照: 実用レベルで備えている(「できなくはない」は含めない)
2. **クッションノード** — デフォルトから外れる関係は、関係を実体化したノードを 1 つ挟んで表現します。例: `unity → written-in-cpp → cpp`(実装言語)、`unity → runs-on-windows → windows`(開発環境)。すべての関係をクッションで包むとグラフが関係ノードだらけになるため、デフォルトから外れるものだけに使います。
3. **橋渡しノード** — 2 つの既存ノードを繋ぐ存在は、両者を参照するノードとして表現します。例: `bevy_rapier → bevy, rapier`、`uefn → unreal-engine, fortnite, ugc`。
4. **追加基準** — 新しいノードは、既存ノードと 2 箇所以上繋がる場合に追加します(孤島を作らない)。ライブラリ層は、実際に使った・検討したものに限ります。網羅は目指しません。これは自分の知識地図です。
5. **限定的なサポートは含めない** — ベータ版・限定提供などは裸参照にせず、表現したくなったら関係をクッションノードで明示します。

## ビューア

`strune.html` をブラウザで開くだけで動きます。サーバ不要・依存なし。

- ノードをクリック: そのノードが新しい中心になる
- 戻るボタン / 足跡: 遷移履歴を巻き戻す
- 検索ボックス: ラベルでノードへジャンプ
- 中央カード: 説明文と入出エッジ数を表示。左右のノードはホバーで説明文をツールチップ表示

## ビルド

Bun を使います(Node 不要)。

```sh
bun install        # 初回のみ
bun run dev        # 開発サーバ(HMR)
bun run build      # 型チェック → Vite ビルド → strune.html を生成
```

`bun run build` は `data.json` を取り込み、JS/CSS を単一の `strune.html` にインライン化します(`vite-plugin-singlefile`)。成果物は依存ゼロで `file://` から直接開けます。
